## 0. 한 줄 요약

가이드(Docusaurus) 페이지에서 지금은 **Storybook을 iframe으로 띄워** 인터랙티브 예제를 보여주는데,
이걸 없애고 **scaffold의 실제 컴포넌트 소스를 가이드에서 직접 import해 렌더링**하도록 바꾼다.
가장 큰 관문은 **Tailwind 4를 Docusaurus 빌드에서 돌리는 것** 하나다. 이것부터 POC로 검증한다.

## 1. 배경 / 동기 (왜 하는가)

- 이 가이드 프로젝트는 나중에 **폐쇄망 SI 프로젝트에 반입**되어, 내부에서 수정·빌드 후
  사내 개발 웹서버에 올려 각 업무 개발자가 참고하는 용도로 쓸 예정.
- 현재 예제는 대부분 **별도로 띄운 Storybook 사이트를 iframe으로 임베드**하는 방식.
  → 폐쇄망에서 Storybook까지 별도 빌드·배포·운영하면 **관리 포인트가 너무 많다.**
- 목표: Storybook 의존을 제거하고, **scaffold 소스를 단일 소스(single source)** 로 삼아
  가이드 안에서 직접 인터랙티브 예제를 렌더링한다.

## 3. 목표 아키텍처 (결정 사항)

### 3.1 소스 공유 방식
- guide의 webpack에 **alias**를 걸어 형제 폴더 scaffold의 **원본 `.tsx`를 직접 참조**한다.
  - scaffold에 `exports` 필드가 없고 내부 경로(`shared/lib/shadcn/ui/...`)로 깊이 들어가야 하므로,
    빌드된 패키지가 아니라 **원본 소스를 guide webpack이 직접 컴파일**하는 구조가 자연스럽다.
  - 예: alias `@scaffold` → `../react-app-scaffold/src`
- 최종적으로는 **npm workspaces 모노레포**로 묶어 **React 중복 인스턴스 문제를 제거**한다(3.4 참고).

### 3.2 Tailwind 4를 Docusaurus에 이식 (이 작업의 유일한 실질 관문)
- `@tailwindcss/postcss`(Tailwind 4의 PostCSS 플러그인)를 Docusaurus PostCSS 파이프라인에 등록.
  - Docusaurus 커스텀 플러그인에서 `configurePostCss(postcssOptions)` 훅으로 플러그인 추가.
- scaffold의 **전역 CSS**(= `@theme`, design-token CSS 변수, `tw-animate-css`, `animate-accordion-*` 등)를
  guide 전역 CSS로 가져와 로드. (Docusaurus는 `docusaurus.config` 의 `theme.customCss` 또는 전역 import)
- Tailwind content 스캔 경로에 **`../react-app-scaffold/src`** 포함 → scaffold 클래스가 purge되지 않게.

### 3.3 재사용 MDX 패턴
- guide에 `<LiveExample>` 래퍼 컴포넌트 1개 생성 (테두리/배경/여백 등 예제 프레임).
- Docusaurus는 **SSR**을 하므로, radix/base-ui 인터랙티브 예제는 `@docusaurus/BrowserOnly`로 감싸
  SSR·portal·`useId` 등 이슈를 회피.
- 사용 예(전환 후 MDX):
  ```mdx
  import BrowserOnly from '@docusaurus/BrowserOnly';
  import LiveExample from '@site/src/components/LiveExample';

  <BrowserOnly>
    {() => {
      const { Accordion, AccordionItem, AccordionTrigger, AccordionContent } =
        require('@scaffold/shared/lib/shadcn/ui/accordion');
      return (
        <LiveExample>
          <Accordion type="single" collapsible>
            <AccordionItem value="a">
              <AccordionTrigger>제목</AccordionTrigger>
              <AccordionContent>내용</AccordionContent>
            </AccordionItem>
          </Accordion>
        </LiveExample>
      );
    }}
  </BrowserOnly>
  ```

### 3.4 모노레포 (최종 단계, 첫 단추 아님)
- 목적: **React 19 단일 인스턴스 보장**(형제 폴더를 alias만으로 참조하면
  각자 `node_modules/react`를 가져 "Invalid hook call / two copies of React" 위험). + 버전 정렬 + 설치 일원화.
- npm workspaces 채택(둘 다 이미 npm). 제안 구조:
  ```
  react-workspace/                 ← 새 루트, workspaces: ["packages/*"], lock 1개
  ├─ packages/
  │  ├─ react-app-scaffold/         (Vite, 컴포넌트 원본)
  │  └─ react-app-scaffold-guide/   (Docusaurus, deps에 "@axiom/react-app-scaffold": "workspace:*")
  ```
- git 재구성 방법(택1):
  - (a) **새 루트 저장소 + 둘을 `git subtree`로 흡수** — 히스토리 보존, 권장.
  - (b) guide를 scaffold 안으로 이동 후 히스토리 병합.
  - (c) 루트 workspace + 둘을 **git submodule** 유지 — 히스토리 안 섞이나 폐쇄망 반입이 번거로움.
- **주의:** 모노레포는 import 편의보다 **React 중복 제거**가 진짜 값어치다.
  import 편의 자체는 webpack alias만으로도 이미 해결된다. **Tailwind 관문은 모노레포로도 안 풀린다.**

## 4. 작업 순서 (이 순서를 지킬 것)

> 원칙: **가역적인 것 먼저, 되돌리기 힘든 것(git 재구성) 나중.**
> Tailwind-in-Docusaurus가 성립하는지가 전체 성패를 가르므로 그것부터 검증한다.

### STEP 1 — Tailwind POC (가역적, guide 내부에서만) ★ 최우선
목표: guide 안에서 Accordion이 **색·애니메이션까지 살아서** 렌더되는지 검증.

1. guide에 런타임 의존 추가: `radix-ui`, `lucide-react`, `clsx`, `tailwind-merge`
   (버전은 scaffold `package.json`과 맞출 것 — 2.2 참고).
2. guide에 Tailwind 4 도입:
   - `tailwindcss`, `@tailwindcss/postcss` 설치.
   - Docusaurus 커스텀 플러그인 작성 → `configurePostCss`에 `@tailwindcss/postcss` 등록.
   - Tailwind 진입 CSS 작성 + content에 `../react-app-scaffold/src/**/*.{ts,tsx}` 포함.
3. scaffold 전역 CSS/테마 이식:
   - scaffold의 Tailwind 진입 CSS(전역 `@theme`, design-token CSS 변수 정의, `tw-animate-css`,
     `animate-accordion-down/up` keyframe)를 찾아 guide 전역 CSS로 가져오기.
     - scaffold의 전역 CSS 진입점 위치를 먼저 확인할 것(예: `src/**/*.css`, `main.tsx`의 css import).
       (조사 시점엔 `src/index.css`가 표준 위치엔 없었음 — 실제 import 지점을 `main.tsx`에서 역추적.)
   - design-tokens는 빌드 산출물(CSS 변수)이므로, scaffold에서 `npm run build:tokens` 후 산출 CSS를 확보.
4. **임시로 Accordion 소스를 guide에 복사**하거나 alias 하나만 걸어서, 한 페이지에서 `<LiveExample>`로 렌더.
   - `@docusaurus/BrowserOnly`로 감쌀 것.
5. **판정:** 접기/펼치기 애니메이션 + 테두리/포커스링/아이콘 색까지 정상 → 이 방식 성립. 다음 단계로.
   - 색이 안 먹으면 → design-token CSS 변수 누락. 애니메이션 안 되면 → keyframe/variant 누락. 3번 보강.

### STEP 2 — 소스 공유 정식화 (alias 또는 모노레포)
- POC 성공 시, 임시 복사본을 제거하고 정식 참조로 교체.
- **모노레포로 갈지 여기서 결정** (4-STEP1이 성공해야 의미 있음):
  - 모노레포 O: npm workspaces 루트 구성(3.4) → `workspace:*` + alias.
  - 모노레포 X(우선 최소): guide `docusaurus.config` webpack alias `@scaffold`→`../react-app-scaffold/src`.
    단, **React 중복** 반드시 확인(guide webpack `resolve.alias`에서 `react`,`react-dom`을
    guide 쪽 하나로 강제 고정하면 중복 회피 가능).

### STEP 3 — 재사용 패턴 확정 & 일괄 전환
- `<LiveExample>` 컴포넌트 확정(코드 표시 토글 등 필요 시 추가).
- Storybook iframe 쓰는 **12개 MDX**를 순차 전환:
  1. accordion, 2. alert, 3. badge, 4. button, 5. button-group, 6. calendar,
  7. carousel, 8. checkbox, 9. combobox, 10. dialog, 11. input, 12. sonner
  (정확 목록은 `grep -rl "AutoHeightStorybookIframe" docs`로 재확인)
- 전환 완료 후 제거 대상: `AutoHeightStorybookIframe.tsx`, `doc-config.ts`의 `STORYBOOK_URL`,
  (scaffold 측) Storybook 관련 스크립트/의존을 유지할지는 별도 결정(단위테스트에서 addon-vitest 사용 여부 확인 후).

### STEP 4 — 폐쇄망 반입 검증
- 오프라인에서 루트 `npm install`(또는 offline mirror) → guide `build` → 정적 산출물 확인.
- baseUrl `/axiom/react-guide/` 유지 확인, 사내 웹서버 경로 정합 확인.

## 5. 리스크 / 주의점 체크리스트

- [ ] **React 중복 인스턴스** — 가장 흔한 실패. 모노레포 호이스팅 또는 webpack `resolve.alias`로 react 단일화.
- [ ] **Tailwind purge** — content 경로에 scaffold `src` 미포함 시 클래스가 사라짐(스타일 전무).
- [ ] **design-token CSS 변수 누락** — 색이 검정/투명으로 나옴. `build:tokens` 산출 CSS 반드시 로드.
- [ ] **커스텀 애니메이션 keyframe** — `animate-accordion-down/up` 미이식 시 펼침 애니메이션 없음.
- [ ] **SSR** — radix/base-ui portal·`useId`. `<BrowserOnly>`로 감쌀 것.
- [ ] **Tailwind preflight 충돌** — Tailwind base reset이 Docusaurus(infima) 기본 스타일과 충돌 가능.
      preflight 범위를 예제 컨테이너로 한정하거나 scope 전략 검토.
- [ ] **버전 스큐** — guide React 19.0 vs scaffold 19.2 등. 모노레포로 정렬 권장.
