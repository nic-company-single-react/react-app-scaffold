# SmartTable — 선언형 고수준 데이터 그리드 모듈

## Context

이 프로젝트는 "어느 사이트에 투입되든 빠르게 시작"하기 위한 React 스캐폴드다. 이미 `$util` / `$ui` / `$router` 라는 **"window-global + imperative + Promise"** 철학으로 보일러플레이트를 한 줄로 줄였다. PC 업무 화면(관리자/그리드)의 가장 큰 마찰은 **데이터 테이블**인데, 현재 [DataTable.tsx](src/domains/example/components/ui-components/DataTable.tsx)는 개발자가 매번 (1) `ColumnDef[]`를 손으로 짜고 (2) 서버 페이징/정렬/검색을 직접 배선하고 (3) 셀 포맷팅(`$util.number/date/string`)을 컬럼마다 반복해야 한다. 또한 색상이 `bg-gray-50` 등으로 하드코딩돼 사이트별 교체가 어렵다.

**목표:** 컬럼을 "설정 맵"으로 선언하면 포맷·정렬·페이징·검색·로딩/빈상태·행액션·export까지 흡수하는 `<SmartTable>`을 만든다. 기존 `$util`을 셀 포맷 엔진으로 재사용하고, 클라이언트 배열/서버 endpoint 양쪽을 한 API로 지원하며, 시맨틱 토큰 기반으로 사이트별 룩 교체가 쉬워야 한다.

## 결정 사항 (사용자 확정)
- **v1 범위:** 코어(컬럼 DSL + `format` 자동포맷 + 정렬/검색/페이지/컬럼토글/행선택 + 클라이언트·서버 모드) **+ 행 액션 메뉴 + 엑셀 export + 일괄 액션 + ref 명령형 핸들** 모두 포함.
- **배치:** `src/shared/smart-table/` 전용 폴더.
- **기존 DataTable:** 공존 + `@deprecated` 표기 (제거하지 않음).
- **전역 `$grid`:** v1 미등록. 인스턴스별 `ref` 핸들로 대체(테이블은 한 화면에 여럿 → 전역 싱글톤 부적합). 수요 확인 후 v2 재고.
- **Export:** 의존성 추가 없이 **UTF-8 BOM CSV**(엑셀이 그대로 열림)를 기본 제공. 진짜 `.xlsx`는 `exporter` prop escape hatch로 위임.

## 공개 API (types.ts)

### 컬럼 DSL
```ts
export type SmartAlign = 'left' | 'center' | 'right';
export type SmartFormat =
  | 'money' | 'number' | 'percent'
  | 'date' | `date:${string}` | 'datetime' | 'fromNow'
  | 'phone' | 'rrn' | 'bizno' | 'maskName' | 'maskMobile' | `mask:${string}`;

export interface ISmartColumn<TRow = Record<string, unknown>> {
  label: string;                       // 헤더 라벨 (필수)
  format?: SmartFormat;                 // 값 자동 포맷 → $util 매핑
  align?: SmartAlign;                   // 숫자=right 권장
  width?: number | string;
  sortable?: boolean;                   // 기본 true
  hideable?: boolean;                   // 컬럼토글 노출 (기본 true)
  hidden?: boolean;
  badge?: { map?: Record<string, { label?: string; variant?: BadgeVariant }>; };
  accessor?: (row: TRow) => unknown;
  className?: string; headerClassName?: string;
  cell?: (ctx: { value: unknown; row: TRow; index: number }) => React.ReactNode; // escape hatch
  columnDef?: Partial<ColumnDef<TRow>>;  // 최종 탈출구
}
export type SmartColumns<TRow = Record<string, unknown>> = {
  [key: string]: string | ISmartColumn<TRow>;   // '라벨' 단축형 또는 상세객체
};
```

### 데이터 바인딩
```ts
export type SmartSelect<TRaw, TRow> = (raw: TRaw) => { rows: TRow[]; total: number };
export interface ISmartServerParamMap {        // 서버마다 파라미터명 다름 → 커스터마이즈
  page?: string; size?: string; sort?: string; order?: string; keyword?: string; pageBase?: 0 | 1;
}
```

### 메인 Props + 핸들
```ts
export interface ISmartTableProps<TRow, TRaw = unknown> {
  columns: SmartColumns<TRow>;
  // 데이터 소스: data XOR endpoint (둘 다면 endpoint 우선 + dev 경고)
  data?: TRow[];
  endpoint?: string;
  select?: SmartSelect<TRaw, TRow>;           // 미지정 시 defaultSelect 추론
  paramMap?: ISmartServerParamMap;
  params?: Record<string, string | number | boolean>;  // 고정 파라미터
  method?: THttpMethod;
  // 행 식별/선택
  rowKey?: keyof TRow | ((row: TRow) => string);       // 기본 'id'
  selectable?: boolean | 'single' | 'multiple';
  onSelectionChange?: (rows: TRow[]) => void;
  // 페이징/정렬/검색
  pageSize?: number; pageSizeOptions?: number[];
  searchable?: boolean; searchPlaceholder?: string; searchKeys?: (keyof TRow)[];
  // 스타일
  variant?: 'card' | 'minimal' | 'bordered';  // 기본 card
  density?: 'compact' | 'normal' | 'loose';    // 기본 normal
  className?: string; classNames?: ISmartTableSlotClassNames;  // 슬롯별 오버라이드
  // 상태/이벤트
  loading?: boolean; emptyText?: React.ReactNode; onRowClick?: (row: TRow, i: number) => void;
  // 확장 슬롯
  toolbarStart?: React.ReactNode; toolbarEnd?: React.ReactNode;
  renderRowActions?: (row: TRow) => React.ReactNode;     // 행 액션 메뉴
  bulkActions?: { label: string; onClick: (rows: TRow[]) => void; variant?: BadgeVariant }[]; // 일괄 액션
  exportable?: boolean | { filename?: string };           // CSV export
  exporter?: (rows: TRow[], columns: SmartColumns<TRow>) => void;  // .xlsx 등 커스텀
}
export interface ISmartTableHandle<TRow> {
  refetch(): void; getSelectedRows(): TRow[]; clearSelection(): void;
  setKeyword(kw: string): void; goToPage(page: number): void;
}
export function defineColumns<TRow>(c: SmartColumns<TRow>): SmartColumns<TRow> { return c; }
```

## 데이터 바인딩 설계

- **모드 분기:** `endpoint ? 'server' : 'client'` ([useSmartTable.ts](src/shared/smart-table/core/useSmartTable.ts)).
- **클라이언트 모드:** `data`를 useReactTable에 주입, 정렬/검색/페이지를 tanstack의 `getSortedRowModel/getFilteredRowModel/getPaginationRowModel`로 처리(기존 DataTable과 동일).
- **서버 모드** ([useServerData.ts](src/shared/smart-table/core/useServerData.ts)): 내부 `{page,size,sort,keyword}` state → paramMap으로 서버 네이밍 변환 → [useApi](src/core/hooks/use-api.ts)의 `params`로 결선. **params 변경 시 queryKey가 바뀌어 자동 refetch**(검증된 동작)되므로 setState→재조회 단방향 흐름. tanstack은 `manualPagination/Sorting/Filtering: true` + `pageCount = ceil(total/size)`. 검색은 300ms debounce 후 `setKeyword` + `setPage(0)`.
- **응답 어댑터:** `select` 미지정 시 `defaultSelect`가 흔한 형태를 순차 추론 — `rows = raw.content ?? raw.rows ?? raw.list ?? raw.items ?? raw.data ?? raw(배열)`, `total = raw.totalElements ?? raw.totalCount ?? raw.total ?? raw.count ?? rows.length`. dev에서 추론 사용 시 1회 경고. (표준 `ApiResponse`엔 페이징 메타가 없으므로 어댑터가 핵심.)

## format → $util 매핑 (formatRegistry.ts)
| format | 매핑 |
|---|---|
| `money` | `$util.number.comma` + 통화 |
| `number` / `percent` | `$util.number.comma` / 백분율 |
| `date` / `date:FMT` / `datetime` / `fromNow` | `$util.date.format(v, FMT)` 등 |
| `phone` / `rrn` / `bizno` | `$util.string` 포맷터 |
| `maskName` / `maskMobile` / `mask:s-e` | `$util.string` 마스킹 |

`cell` 지정 시 format/badge 무시(완전 커스텀). `$util` 진입점 = [src/core/utils/util/index.ts](src/core/utils/util/index.ts).

## 스타일 교체 전략 (smart-table.variants.ts)
[DataTable.tsx](src/domains/example/components/ui-components/DataTable.tsx)의 하드코딩을 시맨틱 토큰으로 치환 → 사이트별 `themes/theme-[project].css` 교체만으로 룩 변경:

| DataTable 하드코딩 | SmartTable 토큰 |
|---|---|
| `bg-gray-50 dark:bg-gray-800/60` | `bg-muted/50` |
| `text-gray-600 dark:text-gray-400` | `text-muted-foreground` |
| `border-gray-200/100 dark:...` | `border-border` |
| `hover:bg-gray-50/80 dark:...` | `hover:bg-muted/50` |
| `data-[state=selected]:bg-blue-50/60` | `data-[state=selected]:bg-accent` |
| `text-gray-700 dark:text-gray-300` | `text-foreground` |

cva variant(`card|minimal|bordered`) + density(`compact|normal|loose`) + 슬롯 className 머지 순서 `cn(variants(...), classNames?.[slot], inline)` (twMerge로 호출측 우선). cva 레퍼런스 = [badge.tsx](src/shared/lib/shadcn/ui/badge.tsx). 배지는 기존 `badgeVariants` 재사용.

## 확장성 (v2 이후 plugins/)
v1 코어는 슬롯(`toolbarStart/End`, `renderRowActions`, `bulkActions`) + `columnDef` escape hatch로 확장. 추후 **고정컬럼 → 행확장 → 가상스크롤 → 트리**는 `plugins/` 모듈 + `plugins` prop(타입 자리만 예약)으로 비파괴 추가. v1엔 미구현.

## 구현 순서
1. **타입 + 포맷 레지스트리** — `types.ts`, `format/*`. $util 매핑 확정(순수 레이어, 먼저 고정).
2. **컬럼 변환** — `core/resolveColumns.ts` (DSL→ColumnDef + CellFormatter).
3. **cva variants + parts** — `smart-table.variants.ts`, `parts/*`. DataTable 하드코딩을 토큰으로 옮기며 재작성.
4. **클라이언트 모드 메인 (MVP)** — `useSmartTable.ts`(client) + `SmartTable.tsx`. 여기까지 기존 DataTable 기능 동등 + DSL/스타일 개선.
5. **서버 모드** — `core/useServerData.ts` + defaultSelect + paramMap + manual* + 스켈레톤/에러.
6. **행 액션 + 일괄 액션 + export + ref 핸들** — `parts/SmartTableRowActions.tsx`, `export/exportCsv.ts`, `useImperativeHandle`.
7. **데모/등록** — 아래 통합 지점.

## 검증
- `npm run dev` 후 `/#/ui-components/smart-table` 데모에서:
  - 클라이언트 모드: 정렬/검색/페이지/컬럼토글/행선택, format(money/date/phone/mask)·badge 렌더 확인.
  - 서버 모드: mock endpoint(JSONPlaceholder 등)로 페이지 이동/정렬/검색 시 네트워크 요청 파라미터 변화 + 자동 refetch 확인. `select` 어댑터로 응답 매핑.
  - export 버튼 → CSV 다운로드(엑셀에서 한글 안 깨짐=BOM 확인). 일괄 액션 = 선택행 콜백. ref 핸들 = 외부 버튼 refetch.
  - variant/density 스위처로 룩 변경, `.dark` 토글 시 토큰 색 정상.
- `npm run build` 타입 통과 (제네릭 추론 `defineColumns`).
- 기존 DataTable 데모(`/#/ui-components/data-table`) 정상 — 회귀 없음.
